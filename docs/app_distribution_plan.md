# UFO Desktop - アプリ化検討資料

作成日：2026-04-07

---

## 概要

UFO Desktop を「自分専用 AI メニューバーアプリ」として発展させるための検討メモ。
RunCat のようなメニューバー常駐型を想定。

---

## Mac アプリとして配布する場合

### 配布方法の選択肢

| 方法 | 難易度 | App Store | 配布形式 |
|------|--------|-----------|----------|
| **py2app** でバンドル | 低 | ❌ 難しい | .app（直配布・GitHub等） |
| **Nuitka** でコンパイル | 中 | △ | .app（直配布） |
| **Swift で書き直し** | 高 | ✅ | App Store / 直配布両対応 |

- 個人利用・GitHub 配布なら **py2app** が現実的
- App Store に出すなら **Swift 書き直し**が必要
- 現状は「個人専用ツール」として育てる方向で OK

---

## 「UFO × ローカル LLM」方向性の検討

### コンセプト軸

RunCat は「CPU 使用率を猫の走るスピードで可視化」という 1 つの軸が明確。
UFO も同様に軸を絞ることで、キャラクター性と実用性を両立させる。

---

### 案1：メニューバー常駐ローカル LLM ランチャー

**UFO = Ollama の「フロントエンド」として振る舞う**

- メニューバーから使いたいモデルをワンクリックで切り替え
- UFO がクイック入力窓（ホットキー起動）になる
- 使用中は UFO がぐるぐる → 終わったら結果をポップアップ表示
- **ポジション**：Alfred / Raycast の「AI 版」をローカルでやる

---

### 案2：スクリーンコンテキスト AI ⭐ 推し

**UFO = 「今見ている画面」を理解して助けてくれる存在**

- ホットキー → 今の画面をキャプチャ → Ollama の vision モデルで解析
- 「このエラーどういう意味？」「このフォームどう書く？」をローカルで
- 既に OCR（glm-ocr via Ollama）が入っているので延長線上で作れる
- 使えるモデル：`llava`、`moondream`、`llava-phi3` など
- **ポジション**：Copilot for Mac をオフライン・無料でやる

---

### 案3：Ollama モデル監視 + 実行ダッシュボード

**UFO = Ollama の「状態モニター」として振る舞う**

- インストール済みモデルの一覧・サイズ・最終使用日をメニューに表示
- モデルのロード状態（GPU VRAM 使用量）を UFO の動きで可視化（RunCat 的）
- `ollama run <model>` を GUI から起動・停止
- **ポジション**：Ollama に GUI を付ける

---

### 案4：クリップボード AI ⭐ 推し

**UFO = コピーしたものを即 AI 処理する**

- テキストをコピー → UFO をクリック → Ollama で要約・翻訳・説明
- コードスニペットをコピー → UFO がレビューして吹き出し表示
- **ポジション**：PopClip の AI 版をローカルで

---

## 推奨方針：案2 ＋ 案4 の組み合わせ

| 理由 | 詳細 |
|------|------|
| 実装コスト低 | 既存の OCR・スクリーンショット機能の延長で作れる |
| UFO らしい | 「UFO が画面を見て答えてくれる」というキャラクター性と一致 |
| 省エネ | ホットキー起動時だけ推論 → 常時負荷ゼロ |
| ローカル完結 | Ollama の vision モデルでオフライン動作 |

---

## 今後の方向性メモ

- **自分専用に特化**して育てる（公開・配布よりも実用優先）
- 省エネ・バックグラウンド負荷ゼロを維持しながら機能追加
- Ollama でダウンロードしたモデルをメニューから簡単に切り替えられる仕組み
- Phase 3（Claude API 連携）も引き続き検討
